# CV fixture provenance — Phase 7.D

Every binary file under `fire/` and `person_aerial/` MUST have a row in
the matching table below: source URL + license + sha256. CI's
`make audit-cv-integrity` re-verifies the sha256s offline. The audit
fails closed if a file appears in the directory but is missing from
this manifest.

License policy:

- Only **CC0 / public-domain** or **explicit BSD/CC-BY (with attribution
  preserved here)** content may be committed.
- **No FLAME, D-Fire, VisDrone, COCO, or ImageNet redistribution.** Those
  datasets are research-only; samples are fetched on demand into the
  gitignored `.cache/cv/samples/` (see `manifest.json`).
- **No real-person photos** unless the subject's CC0 release explicitly
  permits commercial use. Aerial silhouettes / mannequins / synthetic
  placeholders only for the `person_aerial/` bucket.

## "SwarmOS synthetic" — what that means

The current set is a 32x32 zero-pixel PNG with a `tEXt` comment chunk
that makes each file's sha256 distinct. They are generated by
`./_generate.py` (stdlib `zlib` only — no Pillow / numpy required) and
SwarmOS dedicates them to the public domain (CC0-1.0).

The synthetic placeholders satisfy the seam (`CVPerception` picks a
fixture deterministically per ignition; YOLOv8 runs inference; the
top-1 detection's confidence flows into the bus) but they are NOT
representative imagery. Before the 7.D demo:

1. Pull real CC0 frames from Pexels / Unsplash (≤ 6-10 per kind, ≤ 200
   KB each so the committed total stays below ~1 MB).
2. Replace the synthetic file in place (keep the `fixture_NNN.png`
   filename so tests stay stable).
3. Update the matching row above with the source URL + license + the
   new sha256.
4. Run `make audit-cv-integrity` to confirm the sha256 chain.

If you ever need to add a brand-new file (not a replacement), append a
row here BEFORE committing the binary — the audit will fail otherwise.
